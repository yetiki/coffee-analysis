# coffee-analysis

## Description
This project provides a data analysis tool designed to help Dr. Cappuccino select the best coffee suppliers for his shop. 
The report is presented as an interactive **Shiny for Python** application. The app generates a table representing the top 5 coffees based on a custom scoring algorithm. This table is interactive and filters can be applied.
It also produces multiple visual plots of worldwide coffee suppliers to aid in decision-making.

Plots:
    - **Chart 1 — The Main Evidence Plot (Scatter + Bubble)**: Plot every country aggregated: avg `cupper_points` on X, avg `yum_score` on Y, bubble size = number of entries (proxy for producer volume), colour = % of washed processing. This puts all 4 client criteria in one view, and the top 3 countries get annotated labels. This is the hero visual.
    - **Chart 2 — Processing Breakdown for Top 3 Countries (Stacked Bar)**: A grouped/stacked bar chart showing the processing method distribution (Washed, Natural, Pulped Natural, Semi-Washed) side by side for the top 3 recommended countries. This gives the client direct evidence that the washed preference is well-supported.
    - **Chart 3 — Top 3 Country Profile Radar Chart**: A radar/spider chart with one polygon per country, axes being: avg `cupper_points`, avg `yum_score`, % washed, producer count (normalised). This is the clean, client-friendly "at a glance" summary that makes the final recommendation obvious.
    - World map where each country containing a suplier is colored. If the main table is filtered the map will reflect this. color intensity is proportional to the number of supliers in the country. 

The plot is generated using data from the [Coffee Quality Institute](https://database.coffeeinstitute.org/), assembled by Data Scientist James LeDoux into the [Coffee Quality Database GitHub Repository](https://github.com/jldbc/coffee-quality-database).
A score is generated by combining each of the values for aroma, flavour, body, and uniformity into a `YumScore`. Each of the inputs have individual weightings based on the preference of Dr. Cappuccino.


## Pre-requisites
The software dependencies are listed in `requirements.txt`, and should be installed with pip.

## Installation
Clone this git repository and install the dependencies:
```bash
git clone https://github.com/yourusername/coffee-analysis.git
cd coffee-analysis
pip install -r requirements.txt
```

## Usage
Run the Shiny app report using:
`python analyse-coffee.py`

The application will start a local server and should open automatically in your web browser. If not, follow the link provided in the terminal (usually `http://127.0.0.1:8000`).

Or optionally with the additional variables:
`python analyse-coffee.py location=UK minimum_kg=100`

List of available additional variables:
- `location`: Filter by country of origin (e.g., 'Brazil', 'Ethiopia').
- `minimum_kg`: Filter by minimum bag weight available from the supplier.

## Running Tests
To run the tests use:
`python tests.py`

A good output will look like:
"all good!"
